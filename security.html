uv 利用時のセキュリティ運用

uv 利用時に必要な脆弱性監査、ライセンス確認、依存追加時のレビュー観点を整理します。

ページ概要

このページの目的は、uv で依存関係を管理するプロジェクトに対して、脆弱性監査、ライセンス確認、依存追加時レビューをどのように組み込むかを整理することです。監査ツールの導入例と、レビューで確認する観点を扱います。

1. 最低限入れるチェック

項目ツール例目的
脆弱性監査pip-audit既知 CVE や advisories の検出。
ライセンス監査pip-licenses利用不可ライセンスや不明ライセンスの検出。
静的解析ruff, mypy品質と型の回帰検出。

2. pip-audit

powershell
uv add --dev pip-audit
uv sync --locked
uv run pip-audit

3. pip-licenses

powershell
uv add --dev pip-licenses
uv run pip-licenses --format=markdown --output-file=licenses.md

4. 依存追加時のレビュー観点

  • 依存名が社内標準や既存依存と重複・類似していないか。
  • typosquatting らしい名前ではないか。
  • 公開直後のパッケージや、メンテナ変更直後のパッケージではないか。
  • uv.lock の差分が依存追加の目的に対して妥当か。
