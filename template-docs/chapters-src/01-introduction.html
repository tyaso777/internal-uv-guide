Warning
Progressive enhancement

The Python runner, Mermaid, MathJax, Vega-Lite, Prism, and copy buttons are JavaScript enhancements. The sidebar contents, heading references, and previous/next chapter navigation are generated at build time. The sidebar Layout control can switch the page between the standard width and a wide layout; the initial mode is set by layout.defaultMode in the site manifest, and the viewer's last choice is remembered by the browser. This is separate from table-wide, which controls an individual table. Keep important explanations in normal HTML body text so the document remains readable if JavaScript is unavailable.

Inline Math

Use \(...\) for short formulas inside a paragraph. For example, a normal distribution can be written as \(X \sim \mathcal{N}(\mu, \sigma^2)\) using mean \(\mu\) and variance \(\sigma^2\). The standardized value is \(z = (x - \mu) / \sigma\).

def summarize_scores(scores: list[int]) -> dict[str, float]:
    """Returns simple summary statistics for a score list."""
    total = sum(scores)
    count = len(scores)

    return {
        "count": count,
        "average": total / count,
        "maximum": max(scores),
    }


scores = [82, 90, 76, 88]
summary = summarize_scores(scores)
print(summary)

from time import perf_counter


def linear_search_with_count(values: list[int], target: int) -> tuple[int, int]:
    """Finds the target by scanning values and counts comparisons.

    Args:
        values: Sorted integer values.
        target: Value to find.

    Returns:
        A pair of the target index and the number of comparisons.
    """
    comparisons = 0

    for index, value in enumerate(values):
        comparisons += 1
        if value == target:
            return index, comparisons

    return -1, comparisons


def binary_search_with_count(values: list[int], target: int) -> tuple[int, int]:
    """Finds the target using binary search and counts comparisons.

    Args:
        values: Sorted integer values.
        target: Value to find.

    Returns:
        A pair of the target index and the number of comparisons.
    """
    left = 0
    right = len(values) - 1
    comparisons = 0

    while left <= right:
        middle = (left + right) // 2
        comparisons += 1

        if values[middle] == target:
            return middle, comparisons

        if values[middle] < target:
            left = middle + 1
        else:
            right = middle - 1

    return -1, comparisons


def measure_once(n: int, repeats: int) -> tuple[int, int, float, float]:
    """Measures comparison counts and average execution times.

    Args:
        n: Number of elements.
        repeats: Number of repeated searches.

    Returns:
        Linear-search comparisons, binary-search comparisons, average linear-search
        time, and average binary-search time.
    """
    values = list(range(n))
    target = n - 1

    _, linear_comparisons = linear_search_with_count(values, target)
    _, binary_comparisons = binary_search_with_count(values, target)

    start = perf_counter()
    for _ in range(repeats):
        linear_search_with_count(values, target)
    linear_ms = (perf_counter() - start) * 1000 / repeats

    start = perf_counter()
    for _ in range(repeats):
        binary_search_with_count(values, target)
    binary_ms = (perf_counter() - start) * 1000 / repeats

    return linear_comparisons, binary_comparisons, linear_ms, binary_ms


def main() -> None:
    """Runs a simple growth-rate experiment."""
    sizes = [1_000, 10_000, 100_000, 500_000, 1_000_000]
    repeats_by_size = {
        1_000: 200,
        10_000: 100,
        100_000: 20,
        500_000: 5,
        1_000_000: 3,
    }

    print("n          linear_cmp   binary_cmp   cmp_ratio   linear_ms   binary_ms")
    print("-" * 78)

    for n in sizes:
        linear_cmp, binary_cmp, linear_ms, binary_ms = measure_once(n, repeats_by_size[n])
        ratio = linear_cmp / binary_cmp if binary_cmp > 0 else float("inf")
        print(
            f"{n:>9}  "
            f"{linear_cmp:>10}  "
            f"{binary_cmp:>10}  "
            f"{ratio:>9.1f}x  "
            f"{linear_ms:>9.4f}  "
            f"{binary_ms:>9.4f}"
        )

    print()
    print("Interpretation:")
    print("- Comparison counts are the cleanest signal here.")
    print("- Linear search comparisons grow almost directly with n.")
    print("- Binary search comparisons grow slowly because the search range is halved.")
    print("- Timing numbers are useful but noisier than comparison counts.")


main()

Vega-Lite Charts

Vega-Lite renders charts from a JSON specification. When a chapter contains a Vega-Lite chart, the build injects pinned Vega, Vega-Lite, and vega-embed CDN assets so authors can add a chart by editing the JSON spec in the source HTML. Table Examples

Tables are managed as a single block by applying classes and attributes to div.table-wrap. Choose the horizontal behavior first: use the standard table when columns should fit the content width and wrap, or add table-wide when the table should keep its natural width and scroll horizontally. Then choose whether the table also needs a fixed vertical height: add table-fixed-height when many records should scroll under a sticky header. Add table-compact only when shorter row spacing is useful. Use table-grid when full cell borders help matrix-style reading, and data-index-columns to mark leading record-index columns. Numbered table captions are generated for tables such as and .

  • Horizontal width: standard table by default, or table-wide for horizontal scrolling.
  • Vertical height: unrestricted by default, or table-fixed-height for a height-limited table with a sticky header.
  • Row density: regular spacing by default, or table-compact when narrower row spacing is preferred.

A standard table fills the content width and wraps cell text so it usually fits without horizontal scrolling. Fixed-Height Table

Add table-fixed-height when a table has many records and should keep a viewport-aware maximum height. The default height is based on the window size; set --table-max-height on an individual table wrapper when a specific height is needed. The header row stays visible while the body scrolls vertically. This is separate from table-wide; combine both classes only when a table needs horizontal and vertical scrolling.

Operational Notes

Interpreting Runtime Results

Browser execution time is affected by device performance, battery state, browser optimization, and other running processes. Treat timing output as a rough signal for behavior and trends, not as a strict benchmark.

Progressive Reading Design

Body text, links, and code content remain readable as normal HTML. Sidebar contents, heading references, and chapter navigation are present in the generated HTML. Python execution, Mermaid, MathJax, Vega-Lite, Prism, and copy buttons are convenience enhancements added by JavaScript.
