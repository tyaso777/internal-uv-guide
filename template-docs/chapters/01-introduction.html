1. Overview

This page can run Python in the browser with Pyodide. A server-side Python environment is not required, but the first runtime load can take some time because Pyodide is downloaded on demand.

Note
Supplemental context

Use a supplemental note (aside.callout-note) for helpful context, prerequisites, and recommendations. Example: This template is designed to remain readable on narrow screens.

Success
Completed or verified state

Use a success callout (aside.callout-success) for completed checks, verified outcomes, and successful states. Example: The generated chapter passed the HTML validation step.

Warning
JavaScript-dependent enhancements

Use a warning note (aside.callout-warning) for constraints, environment-specific behavior, and exceptions readers should know in advance. Example: Python execution, Mermaid, MathJax, Vega-Lite, and Prism depend on JavaScript; the chapter contents and navigation are generated at build time.

2. Template Usage

This section is a minimal editing guide for creating a new HTML document from the template. It shows the main writing patterns so both human editors and generation models can extend the document consistently. For an interactive code example, see Section 4.2.

Note

Use <section id="..." data-toc data-toc-title="..."> for top-level sections that should appear in the table of contents. For optional subsection entries, add id, data-toc, and data-toc-level="3" to the heading.

Common Components

  • Supplemental notes and recommendations: <aside class="callout callout-note"> with <div class="callout-label">Note</div>
  • Completed checks and verified outcomes: <aside class="callout callout-success"> with <div class="callout-label">Success</div>
  • Warnings and constraints: <aside class="callout callout-warning"> with <div class="callout-label">Warning</div>
  • Optional callout subtitle: <div class="callout-title">...</div>
  • Standard body card: <div class="card">
  • Muted card: <div class="card card-muted">
  • Compact card: <div class="card card-compact">
  • Code display: <pre class="code-block language-python"><code class="language-python">
  • Math: use \(...\) for inline math and \[...\] for display math
  • Mermaid diagrams: write the diagram definition inside <div class="mermaid">.
Warning
Progressive enhancement

The Python runner, Mermaid, MathJax, Vega-Lite, Prism, and copy buttons are JavaScript enhancements. The sidebar contents, heading references, and previous/next chapter navigation are generated at build time. The sidebar Layout control can switch the page between the standard width and a wide layout; the initial mode is set by layout.defaultMode in the site manifest, and the viewer's last choice is remembered by the browser. This is separate from table-wide, which controls an individual table. Keep important explanations in normal HTML body text so the document remains readable if JavaScript is unavailable.

3. Math Examples

These examples show inline and display math rendered by MathJax. When creating a new document, copy this section and replace the content while preserving the display patterns.

3.1. Inline Math

Use \(...\) for short formulas inside a paragraph. For example, a normal distribution can be written as \(X \sim \mathcal{N}(\mu, \sigma^2)\) using mean \(\mu\) and variance \(\sigma^2\). The standardized value is \(z = (x - \mu) / \sigma\).

3.2. Display Math

Use \[...\] for standalone formulas, optionally wrapped in div.math-block. The density definition in Equation 1-1 and the moment identities in Equation 1-2 are numbered examples.

4. Python Code Display and Execution

Use static code blocks for code that readers only need to inspect, and use the browser runner for code readers should edit and execute. This keeps explanatory snippets separate from interactive examples.

4.1. Python Code Block

For code that does not need to run in the browser, combine pre.code-block with code.language-python. Place div.code-caption above the block to show the language name or file name.

python
def summarize_scores(scores: list[int]) -> dict[str, float]:
    """Returns simple summary statistics for a score list."""
    total = sum(scores)
    count = len(scores)

    return {
        "count": count,
        "average": total / count,
        "maximum": max(scores),
    }


scores = [82, 90, 76, 88]
summary = summarize_scores(scores)
print(summary)

4.2. Executable Code Example

Use the runner source block for code that readers should edit and execute. The build script expands div[data-python-runner] into the full browser runner UI.

Warning
Pyodide CDN loading

The Python runner loads Pyodide from a CDN only after the reader presses Load Python Runtime. Pyodide is loaded with importScripts() inside a Web Worker, so normal script-tag SRI with integrity cannot be used there. For documents that require strict supply-chain control, host a reviewed Pyodide distribution locally.

Try changing the input sizes and run the code. On slow connections or constrained devices, the first Python runtime load can take some time.

Output

Press "Load Python Runtime" first. Python will run in a Web Worker.

            Mermaid renders flowcharts and structural diagrams from text definitions.
            You can update a diagram by editing text in the HTML instead of recreating an image.
            Mermaid also supports sequence diagrams, class diagrams, state diagrams, and more.

7. Vega-Lite Charts

          

            Vega-Lite renders charts from a JSON specification.
            When a chapter contains a Vega-Lite chart, the build injects pinned Vega, Vega-Lite, and vega-embed CDN assets so authors can add a chart by editing the JSON spec in the source HTML.
            The example below combines a year selector, clickable stacked bars, tooltips, and a linked component breakdown without adding custom JavaScript.
          


          
vega-lite json excerpt

          
{
  "$schema": "https://vega.github.io/schema/vega-lite/v5.json",
  "params": [
    {
      "name": "selectedYear",
      "value": 2026,
      "bind": {
        "input": "select",
        "options": [2025, 2026],
        "name": "Year "
      }
    }
  ],
  "vconcat": [
    {
      "params": [
        {
          "name": "selectedItem",
          "value": [{"region": "East", "product": "Basic"}],
          "select": {"type": "point", "fields": ["region", "product"]}
        }
      ],
      "mark": "bar",
      "transform": [{"fold": ["New", "Expansion", "Renewal", "Support"], "as": ["component", "amount"]}]
    },
    {
      "transform": [
        {"filter": {"param": "selectedItem"}},
        {"fold": ["New", "Expansion", "Renewal", "Support"], "as": ["component", "amount"]}
      ],
      "mark": {"type": "line", "point": true}
    }
  ]

8. Table Examples

          

            Tables are managed as a single block by applying classes and attributes to div.table-wrap.
            Choose the horizontal behavior first: use the standard table when columns should fit the content width and wrap, or add table-wide when the table should keep its natural width and scroll horizontally.
            Then choose whether the table also needs a fixed vertical height: add table-fixed-height when many records should scroll under a sticky header.
            Add table-compact only when shorter row spacing is useful.
            Use table-grid when full cell borders help matrix-style reading, and data-index-columns to mark leading record-index columns.
            Numbered table captions are generated for tables such as Table 1-1 and Table 1-2.
          

          
    
            
  • Horizontal width: standard table by default, or table-wide for horizontal scrolling.
    • 
            
  • Vertical height: unrestricted by default, or table-fixed-height for a height-limited table with a sticky header.
    • 
            
  • Row density: regular spacing by default, or table-compact when narrower row spacing is preferred.
    •

8.3. Fixed-Height Table

          

            Add table-fixed-height when a table has many records and should keep a viewport-aware maximum height.
            The default height is based on the window size; set --table-max-height on an individual table wrapper when a specific height is needed.
            The header row stays visible while the body scrolls vertically.
            This is separate from table-wide; combine both classes only when a table needs horizontal and vertical scrolling.

9. Operational Notes

          

            
Interpreting Runtime Results

            

              Browser execution time is affected by device performance, battery state, browser optimization, and other running processes.
              Treat timing output as a rough signal for behavior and trends, not as a strict benchmark.
            

          


          

            
Progressive Reading Design

            

              Body text, links, and code content remain readable as normal HTML.
              Sidebar contents, heading references, and chapter navigation are present in the generated HTML.
              Python execution, Mermaid, MathJax, Vega-Lite, Prism, and copy buttons are convenience enhancements added by JavaScript.
