ページを分ける方針

ページは分ける方がよいです。uv は導入時だけ読む情報、日常的に参照するコマンド、移行時だけ必要な判断、セキュリティ監査の運用が混在します。

社内標準として決めるべきこと

  • uv 本体の標準バージョンと更新頻度
  • Python の標準バージョン、プロジェクトごとの固定方法
  • proxy / SSL 設定をユーザー環境変数に置くか、PowerShell profile に置くか
  • 依存解決で最新公開直後のパッケージを避ける日数
  • uv.lock をレビュー対象に含める方針
  • 脆弱性監査とライセンス監査を CI のどこで実行するか
最新公開後 X 日以内のものを避ける、という考え方は pandas など uv で解決・インストールする依存パッケージに対する運用です。uv 本体については X 日ルールではなく、社内で確認済みの標準バージョンを明示する方針にします。
