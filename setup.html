proxy の確認

powershell
netsh winhttp show proxy

$env:HTTP_PROXY="http://proxy.example.co.jp:8080"
$env:HTTPS_PROXY="http://proxy.example.co.jp:8080"

社内 CA / SSL inspection

社内 proxy が TLS 通信を検査している場合、OS の証明書ストアに入っている社内ルート CA を uv に利用させます。

powershell
$env:UV_SYSTEM_CERTS="1"

private package repository

toml
[[tool.uv.index]]
name = "internal"
url = "https://packages.example.co.jp/simple"
default = true

新しすぎるリリースを避ける

依存パッケージの公開直後は、破損した wheel、差し戻し、悪意あるパッケージ混入、社内 mirror への反映遅延が起きる可能性があります。

powershell
uv lock --exclude-newer "7 days"
uv sync --exclude-newer "7 days"

$env:UV_EXCLUDE_NEWER="7 days"
uv lock
uv sync
社内標準として「実行日時から X 日前より後にアップロードされた依存パッケージは使わない」と決めます。例として X=7 の場合は --exclude-newer "7 days" を使います。
