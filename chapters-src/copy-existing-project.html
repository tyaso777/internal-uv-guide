clone・コピーした uv プロジェクトの始め方

uv 対応済みの既存プロジェクトを取得し、lock file を尊重して手元で動かす流れを示します。

ページ概要

このページの目的は、uv 対応済みの既存プロジェクトを clone またはコピーし、手元の端末で同期して動作確認することです。既存の uv.lock を尊重し、意図しない依存更新を避けながら環境を再現します。

プロジェクトを取得する

powershell
git clone https://github.com/example/my-app.git
cd my-app
Note
フォルダをコピーして始める場合も、.venv はコピーせず、手元で uv sync により再生成します。

ネットワーク設定を適用する

依存パッケージ取得時に社内 proxy や社内 CA が必要な場合は、同期前にこの PowerShell セッションへ設定します。

powershell
# proxy をこの PowerShell セッションに設定する
$env:HTTP_PROXY="http://proxy.example.co.jp:8080"
$env:HTTPS_PROXY="http://proxy.example.co.jp:8080"

# OS の証明書ストアをこの PowerShell セッションで利用する
$env:UV_SYSTEM_CERTS="1"

# 設定された値を確認する
echo $env:HTTP_PROXY
echo $env:HTTPS_PROXY
echo $env:UV_SYSTEM_CERTS

Python バージョン指定を確認する

.python-version と pyproject.toml の requires-python を確認し、プロジェクトが想定する Python バージョンで同期します。

powershell
Get-Content .python-version
Select-String -Path pyproject.toml -Pattern "requires-python"

仮想環境の作成先を設定する

共有フォルダ上で作業する場合は、uv sync の前に UV_PROJECT_ENVIRONMENT を設定します。clone またはコピーしたプロジェクトでは、現在いるプロジェクトフォルダ名と保存場所から、このプロジェクト用の仮想環境パスを作ります。

powershell
# プロジェクトフォルダの絶対パスを取得する
$projectRoot=(Resolve-Path .).Path

# プロジェクト名を現在のフォルダ名から取得する
$projectName=Split-Path -Leaf $projectRoot

# 同じプロジェクト名の仮想環境と衝突しないように、
# プロジェクトの保存場所をもとに短いハッシュを作る
$normalizedProjectRoot=$projectRoot.ToLowerInvariant()
$sha256=[System.Security.Cryptography.SHA256]::Create()
$hashBytes=$sha256.ComputeHash([System.Text.Encoding]::UTF8.GetBytes($normalizedProjectRoot))
$hash=([System.BitConverter]::ToString($hashBytes) -replace "-", "").Substring(0,8).ToLowerInvariant()

# 仮想環境の作成先を、このプロジェクト用のローカルフォルダに設定する
$env:UV_PROJECT_ENVIRONMENT="$env:USERPROFILE\.uv\venvs\$projectName-$hash"

# 設定された仮想環境の場所を確認する
echo $env:UV_PROJECT_ENVIRONMENT

環境を同期する

既存の lock file を使って環境を再現します。通常の利用開始時は lock file を更新しません。

powershell
uv sync --locked

動作確認する

powershell
uv run python --version
uv run pytest

lock file 更新の扱い

利用開始だけが目的の場合は uv.lock を更新しません。依存追加やバージョン更新が目的の場合だけ、変更理由を明確にして lock file を更新します。
